**Architecture Patterns**:

1. **Components** (`/components/`):
   - PascalCase naming: `UserCard.tsx`, `NavBar.tsx`
   - Co-locate styles with components
   - Prefer composition over props drilling
   - Props with TypeScript interfaces:
   ```tsx
   interface UserCardProps {
     user: User;
     onSelect?: (id: string) => void;
   }
   
   export const UserCard: React.FC<UserCardProps> = ({ user, onSelect }) => {
     // ...
   };
   ```

2. **Hooks** (`/hooks/`):
   - Prefix with `use`: `useAuth.ts`, `useFetch.ts`
   - Extract complex logic from components
   - Example:
   ```tsx
   export const useCounter = (initial = 0) => {
     const [count, setCount] = useState(initial);
     const increment = useCallback(() => setCount(c => c + 1), []);
     return { count, increment };
   };
   ```

3. **Types** (`/types/`):
   - Use `interface` for component props and extendable shapes
   - Use `type` for unions, intersections, and utility types
   - Export shared types from index files

4. **State Architecture**:
   - **Local State**: `useState` for component-specific state
   - **Derived State**: `useMemo` for computed values (NOT useEffect!)
   - **Global State**: Zustand store for app-wide state
   - **Server State**: React Query for API data with caching

**React Best Practices**:
- Use `useCallback` for event handlers passed to children
- Use `useMemo` for expensive calculations
- Clean up effects in `useEffect` return function
- Never mutate state directly (`setState(prev => [...prev, newItem])`)
- Always provide `key` props in lists (real IDs, not indices)

**Common Pitfalls to Avoid**:
- ❌ Using `useEffect` for derived state (use `useMemo`)
- ❌ Mutating state directly
- ❌ Missing keys in `.map()` loops
- ❌ Using `any` type (use `unknown` and narrow)
- ❌ Prop drilling more than 2 levels (use context or Zustand)
- ❌ useEffect dependency array mismatches

**Testing**:

- E2E: Playwright or Cypress
- Component: Storybook for visual testing

## MARKDOWN FORMATTING STANDARDS

- Use fenced code blocks with language: ` ```python ... ``` `

- Use `render_diffs(file:///path)` shorthand to show all file changes

> Essential requirements or critical information
